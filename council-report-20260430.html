5 advisors · 5 peer reviews · 1 chairman synthesis · 2026-04-30
What agent capabilities and architectural decisions maximize hiring signal for DevTrack as an AI Engineer / MLOps portfolio project targeting Sydney's mid-to-large engineering companies?
Chairman's Verdict

Where the Council Agrees

  • One agent that works flawlessly beats three agents that are impressive on paper.
  • Failure modes are the interview — not the feature list, not the agent count.
  • Hirers probe 3 things: failure modes, cost awareness, cold explainability (10 min, no README).
  • The README and demo video are load-bearing artifacts, not afterthoughts.

Where the Council Clashes

  • CrewAI vs LangGraph: Executor says use both. Contrarian says using both is a red flag to senior engineers — signals tutorial-following without design understanding. Chairman sides with Contrarian: pick LangGraph only.
  • "Living artifact" vs frozen demo: Expansionist says use your real job search data. Three peer reviewers flagged this as high-variance — a live demo that breaks collapses the whole story. Freeze it with realistic seed data. The story is still true.
  • React dashboard timing: Executor says build it last. Outsider says skip Phase 1 entirely. Outsider wins: Swagger UI is more credible to engineering audiences for Phase 1.

Blind Spots the Council Caught

  • Visa filter shapes the target audience: Sydney AI hiring = fintech, govtech, professional services. These orgs value reliability, audit trails, cost-bounded explainable agents — not demo novelty.
  • Fishburners leads; DevTrack supports: Your internship building real agentic pipelines is the stronger credential. Frame DevTrack as "the engineering depth behind what I'm doing at Fishburners."
  • GitHub commit history is a credential: Don't squash commits. Debugging iterations in the history signal engineering process before anyone reads the code.
  • Polyglot = demo risk: Node.js → Python FastAPI → LangGraph = 3 failure boundaries. Build the Python service to be demoable in isolation. The Node.js integration is a bonus.

The Recommendation

Build ONE LangGraph StateGraph (4 nodes: fetch profile → fetch JD → gap analysis → talking points). Add a circuit breaker at gap_analysis. Surface token_cost_usd in every response. Build a /runs/{run_id} endpoint returning full state transition trace. Add a "Failure modes and what I learned" section to the README.

Do NOT build in Phase 1: CrewAI alongside LangGraph · Interview Prep crew · Ollama fallback · React dashboard · RAG pipeline

The One Thing to Do First

Before writing a single line of Python: set a 10-minute timer and write the cold explanation — architecture, key design decisions, what breaks first under load — as if speaking to a senior engineer interviewing you. If you can't write it clearly in 10 minutes, you don't yet know what to build.

Advisor Responses (click to expand)

The Contrarian Contrarian

The fatal flaw nobody is saying: this project is a job tracker that tracks job applications. You are the target user. Recruiters will see a productivity app with an AI wrapper — not an AI Engineer signal.

CrewAI + LangGraph together is a red flag. They solve overlapping problems with different abstractions. Pick LangGraph — it's lower-level and more production-realistic.

"Ollama fallback" admits your architecture doesn't require strong reasoning. The three-crew structure is arbitrary — none of those tasks require multi-agent orchestration.

Build one agent that does one thing provably well. That is harder and more impressive.

First Principles Thinker Principles

Hirers evaluate 3 things: (1) failure modes — circuit breakers, tool call depth limits, hallucination handling; (2) cost awareness — token budget, retry logic, backoff; (3) cold explainability — architecture in 10 min, no README.

You are not building an impressive project. You are building a conversation piece that makes you hireable. Those are different optimization targets.

Don't cargo-cult multi-agent patterns without understanding the tradeoff against simpler chain-of-thought. That's the most common junior AI engineer mistake right now.

The Expansionist Expansionist

Three agents and a dashboard is a 2024 portfolio. The biggest multiplier: instrument everything and surface the trace. Show token costs per run. Build a /runs endpoint.

The Anthropic + Ollama fallback is your MLOps story — document latency, cost, and quality deltas. Ship a comparison table. System design conversation starter.

DevTrack is a living artifact of your own job search. If you get hired partly because of it — that's a case study, not a portfolio project. Set the ceiling higher.

The Outsider Outsider

"DevTrack API" tells me nothing about AI. Fix that first. Rename to "DevTrack: Agentic Job Intelligence API." I should know within 3 seconds.

The demo video is the portfolio. 90-second recording pinned to README. No demo video = project doesn't exist.

The thing that separates you: show a decision the agent made, explain why it was wrong, show how you fixed the prompt or graph edge. The project isn't the code — it's the story you tell about the code.

The Executor Executor

Build in order: (1) Job Research Crew — 2 CrewAI agents, triggered automatically on application creation. (2) LangGraph 4-node StateGraph. (3) React dashboard last — minimal.

Don't build: Interview Prep crew, Ollama fallback, RAG pipeline, Docker Compose with 8 services.

The hiring signal: Node.js API → Python microservice → multiple AI agents → structured JSON. One working demo beats three half-built features. Build the CrewAI crew this weekend.
